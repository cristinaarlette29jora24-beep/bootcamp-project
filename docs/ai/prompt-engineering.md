## Prompt Engineering
La construcción de TaskFlow se ha basado en instrucciones precisas para obtener código limpio y funcional. Se han utilizado los siguientes tipos de prompts:

# 1.  Prompt de Refactorización: 
"Optimiza la función de guardado en LocalStorage para que sea más eficiente, pero mantén la misma estructura de nombres que ya tiene el proyecto".

# 1. Prompt de Depuración (Debug):
 "Revisa el archivo app.js y detecta por qué no se actualiza el contador de tareas completadas al marcar el checkbox".

# 1. Prompt de Estilo: 
"Genera reglas CSS para que la lista de tareas sea totalmente responsive en dispositivos móviles sin usar librerías externas como Bootstrap".

# 1. Prompt de Documentación: 
"Explica línea por línea qué hace la función de filtrado de prioridades para entender mejor la lógica del código".

# 1. Prompt de Seguridad: 
"Valida que los inputs de texto no acepten código malicioso y limpia los datos antes de guardarlos en el sistema".

El uso de estas instrucciones demuestra que la calidad del código generado depende directamente de la claridad y los detalles técnicos aportados en cada petición.
